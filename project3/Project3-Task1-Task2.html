Task 1 — Proposal

Introduction

TaskFlow is a full-stack Single Page Application (SPA) designed to help individuals organize and manage their daily tasks effectively. Users can create, update, prioritize, and complete tasks through a clean and responsive interface.

The app addresses a common problem: the lack of a simple, personal, and secure task management tool that works across devices without requiring a complex setup. TaskFlow delivers a modern web experience with user authentication, real-time data updates, and offline capability through Progressive Web App (PWA) technology.

The project is built as a full-stack application where React powers the front-end and Express + MongoDB powers the back-end, following industry-standard practices for security, scalability, and maintainability.

Task 1 — Proposal

Goals & Objectives

Primary Goals

  • Build a fully functional CRUD SPA for personal task management
  • Provide secure signup, login, and logout with JWT authentication
  • Enable users to add, edit, delete, and complete tasks
  • Support task prioritization (High / Medium / Low)
  • Deliver a responsive UI accessible on mobile and desktop

Technical Objectives

  • Implement React with hooks and Context API
  • Build a RESTful API with Express and MongoDB
  • Secure routes with JWT tokens stored in HTTP-only cookies
  • Apply SASS for modular, maintainable styling
  • Enable PWA for installability and offline support
  • Deploy on a live server with SSL/HTTPS

Target Audience

Primary Users

  • Age: 18–45
  • Gender: All genders
  • Occupation: Students, freelancers, remote workers, professionals
  • Tech comfort: Moderate to high — regular smartphone and web app users
  • Device: Desktop, tablet, and mobile

User Needs

  • A private, personal space to track daily tasks
  • Simple and fast interface — no learning curve
  • Ability to prioritize tasks and filter by status
  • Secure account so data is private to each user
  • Works on any device, even with limited connectivity (PWA)
Task 1 — Proposal

Key Features

  • User Authentication — Secure signup, login, and logout with JWT stored in HTTP-only cookies and server-side sessions
  • Task CRUD — Create, read, update, and delete personal tasks; each task belongs exclusively to the logged-in user
  • Priority Levels — Assign High, Medium, or Low priority to each task with color-coded visual indicators
  • Task Status & Filtering — Mark tasks as complete/incomplete; filter view by All, Active, or Done
  • Form Validation — Client-side and server-side validation on all forms with clear error messages
  • Responsive Design — Bootstrap 5 grid adapts layout for mobile, tablet, and desktop viewports
  • PWA Support — Web App Manifest + Service Worker enable installation on device home screen and offline browsing
  • Live Stats — Real-time counter showing total, active, and completed task counts
